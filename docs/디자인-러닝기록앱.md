# 러닝 기록 앱 — UI/UX 디자인 명세

> **구현 동기화:** 실제 화면·토큰·내비·브랜딩 자산과 맞춰 갱신. 토큰 원본은 [design-tokens.css](./design-tokens.css), **런타임 번들**에는 `app/globals.css` 상단 `:root`가 동일 값으로 인라인 복사됨(`@import` 누락 방지). Tailwind는 `tailwind.config.ts`에서 위 CSS 변수를 참조.

기획서: [기획-러닝기록앱.md](./기획-러닝기록앱.md)

## 1. 디자인 방향

| 키워드 | 적용 |
|--------|------|
| **가볍고 빠름** | 여백 넉넉, 한 화면에 정보 단계적 공개 |
| **신뢰·기록** | 숫자(거리·페이스) 가독성 우선, 차트는 보조 |
| **에너지** | 액센트 컬러로 “움직임” 표현, 과한 그라데이션은 지양 |
| **토이·1주** | 다크 모드 토큰만 준비(F-08), **라이트 기본**·토글 UI는 미구현 |

**톤:** 친근한 설명형 문장. 숫자·날짜 라벨은 짧고 명확하게.

## 2. 정보 구조 · 내비게이션

### 2.1 로그인 후 IA

```
대시보드 (/dashboard) — 요약·목표·연속 기록·주~전체 차트·최근 기록·날씨
랭킹 (/ranking) — 기간·지표별 리더보드
커뮤니티 (/community) — 챌린지·한 줄 피드·댓글·반응
기록 (/runs) — 목록
  └ 신규 (/runs/new)
  └ 수정·삭제 (/runs/[id]/edit)
설정 (/settings) — 닉네임·이모지·주/월 목표, 배지 모음, 로그아웃
```

### 2.2 내비 패턴 (구현)

| 뷰포트 | 패턴 |
|--------|------|
| **모바일 (<768px)** | **상단 고정 헤더(간단):** 좌측 로고·RunLog → `/dashboard`, 우측 **계정 캡슐 버튼** 클릭 시 메뉴(`설정`, `로그아웃`). **하단 고정 탭 4개:** `대시보드` · `랭킹` · `커뮤니티` · `기록`(설정 탭 없음). 콘텐츠 하단은 탭에 가리지 않도록 `pb-20` 여백. **FAB** `+`: 대시보드에서 기록 추가(우하단, `aria-label="기록 추가"`). |
| **태블릿/데스크톱 (md↑)** | **상단 고정 헤더:** 좌측 **`RunLogLogo`(PNG) + RunLog 텍스트** 링크 → `/dashboard` (`aria-label="RunLog 홈"`), **텍스트 네비 링크 4개**(대시보드·랭킹·커뮤니티·기록), 우측 **계정 캡슐**(프로필 이모지 + 「로그인」+ 표시명 truncate) — 클릭 시 **드롭다운**: `설정`(`/settings`), `로그아웃`(세션 종료 후 `/`). 설정 페이지에도 **로그아웃** 버튼 유지. |

구현: 주요 IA는 하단 탭·상단 링크로 이동하고, **설정·로그아웃은 계정 메뉴**에서 접근.

터치 최소 영역: **44×44px** 이상을 목표(버튼·탭·FAB).

## 4. 화면별 와이어프레임 (구현 기준)

### 4.1 랜딩 `/` (비로그인)

- **상단 별도 앱바 없음.** 세로 중앙 정렬 단일 컬럼(max-width 제한).
- 상단 **브랜드 블록:** `RunLogLogo`(세로 기준 약 **56px**) + 액센트색 대문자 **RunLog** 캡션(`text-caption` 계열) → 큰 제목 `오늘 러닝, 가볍게 기록` → 부제 **두 줄**(첫 줄: 거리·시간·메모·차트 안내, 둘째 줄: Google 시작 안내).
- 로고 영역에 **`shadow-card` 미적용**(카드형 그림자 없음).
- **Primary:** `Google로 계속하기` 전폭 버튼.
- `GOOGLE_CLIENT_ID` 등 미설정 시 **호박색 안내 카드**로 `.env` 안내.
- OAuth 오류: URL `error` 파라미터 또는 `/auth/error`에서 메시지.

### 4.2 대시보드 `/dashboard`

- **상단:** 한 줄 **`flex`** · **`items-start`** — 왼쪽에 제목 `대시보드` + `안녕하세요, {이름}님`(`min-w-0 flex-1`), 오른쪽에 **날씨 스냅샷**(`DashboardWeather`, `max-w-[min(100%,16rem)]`, 우측 정렬). 모바일은 날씨 카드 **`px-2.5 py-1.5`**·작은 온도 글자(`text-xl`)·촘촘한 부가 줄로 인사 블록과 **높이 감각을 맞춤**(`sm` 이상은 기존 여백·`text-2xl`). (`DashboardWeatherHeaderSuspended`)  
    - **데이터:** 공공데이터포털 기상청 `VilageFcstInfoService_2.0` — 초단기실황(`getUltraSrtNcst`: 기온·습도·PTY 등) + 초단기예보(`getUltraSrtFcst`: `SKY` 맑음/구름/흐림). 환경 변수 `DATA_GO_KR_SERVICE_KEY` 등(`.env.example` 참고).  
    - **표현:** `PTY`·`SKY` 코드 → `weather-icon-map` + **이모지**(`WeatherIcon`, 외부 이미지 URL 없음). **레이아웃:** 이모지와 온도·문구는 **가로 한 줄**(모바일·PC 동일). 이모지 컨테이너는 **`h-10 w-10`·`text-3xl`** 로 뷰포트 통일. **`네이버 날씨 →`:** `sm` **미만** 은 온도·습도와 **같은 텍스트 열 안**에서 **왼쪽** 정렬; **`sm` 이상** 은 메인 행 아래 **카드 전폭** 줄에서 **`text-right`**. API 오류 문구일 때는 모바일 **`text-left`** · 데스크톱 **`text-right`**. 링크 목적지·표시는 `NAVER_WEATHER_URL` 등 기존과 동일.  
    - **오류 시:** 인증 실패·NO_DATA 등 메시지를 같은 컴팩트 영역에 표시.  
    - **관측 지점 명칭**은 UI에 노출하지 않음(과거 카드형 실험 제거).  
    - **캐시:** 서버에서 격자·관측 슬롯 기준 `unstable_cache`(짧은 TTL, README·기획과 동일)로 외부 API 호출을 줄임 — 표시값이 실시간과 수분 차이 날 수 있음.
    - **로딩:** 날씨 블록은 **`Suspense`** 로 스트리밍(`DashboardWeatherHeaderSuspended`·`DashboardWeatherSkeleton`). 동일 요청 내 **`React.cache(getWeatherSnapshot)`** 로 조회 1회(DB 집계와 외부 API 대기 분리).
- **첫 번째 요약 그리드:** **`grid-cols-1`** 모바일(카드 **전폭** 세로 스택) · **`sm:grid-cols-3`** 데스크톱 — **연속 기록**, **이번 주 목표**, **이번 달 목표**. 날씨는 그리드에 넣지 않음.
- **두 번째 요약 그리드:** **`grid-cols-1`** 모바일 · **`sm:grid-cols-3`** — 선택 기간(`주`·`월`·`년`·`전체`) **횟수**·**거리(km)**·**운동 시간**(합산 초 → `N분 N초`, 없으면 `—`).
- **차트 섹션:** 제목·캡션은 기간에 따라 `이번 주 거리`(요일별) / `이번 달 거리`(일별) / `올해 거리`(월별) / `전체 기간 거리`(월별 합산). 우측(또는 아래) **세그먼트:** `주` | `월` | `년` | `전체` — URL `?range=month` / `year` / `all`(기본 주간은 쿼리 없음). **`월`** 보기: **`sm`(640px) 미만** 에서 **단일 차트 + 가로 스크롤**(일자 간격·0km·X축 정렬 유지; Y축도 스크롤에 함께 이동). **`sm` 이상** 은 스크롤 없음(`DashboardCharts`). **차트 카드 높이:** 주·월 동일(약 260px), 년·전체 약 300px — 상세 §5.4.
- **해당 기간에 런이 한 건도 없으면:** 막대 차트 대신 **점선 카드 안내 문구**만 표시(“기록을 추가하면 차트가 표시됩니다.”).  
  기간에 런이 있으면 **Recharts 막대**. 일별 합계가 모두 0이면 **툴팁 미표시·막대 활성 레이어 없음**(구현: `hasChartData`). 그 외 데이터가 있을 때는 **툴팁만** 표시하고, **회색 호버 밴드(Tooltip cursor)·활성 막대 오버레이(activeBar)** 는 사용하지 않음(`cursor={false}`, `activeBar={false}`).
- **최근 기록:** 최대 5건, 행 클릭 → `/runs/[id]/edit`. 거리·페이스(/km), 선택 시 심박·케이던스 한 줄, 메모 clamp. 모바일 상단 `기록 추가` 링크 + FAB `+`.
- **전체 기록 보기:** `md` 이상에서만 “전체 기록 보기 →” 링크 노출, **해당 줄 우측 정렬**(`flex justify-end`). 모바일은 하단 탭 `기록`으로 이동.

### 4.3 기록 목록 `/runs`

- 제목 `기록` + 우측 `추가` 버튼 → `/runs/new`.
- 빈 상태: 점선 카드 + “아직 기록이 없습니다. 첫 러닝을 남겨 보세요.”
- 각 행: **카드 전체가 링크** → 수정 화면. 날짜, `km · 분·초 · 페이스 …/km`, 심박·케이던스(있을 때), 메모.  
  **행 우측 `⋯` 메뉴는 없음** — 삭제는 수정 화면에서만.

### 4.4 기록 추가·수정 `/runs/new`, `/runs/[id]/edit`

- 폼 카드: 활동일(date), 거리(km), **시간 분 / 초(0–59)** 그리드, **평균 심박·케이던스(선택)** 그리드, 메모 textarea, 저장·취소. 수정 시 **삭제**(Danger, `confirm` 문구: 기획 마이크로카피와 동일 취지).
- 케이던스 보조 설명 캡션(spm).

### 4.5 설정 `/settings`

- 제목 `설정`.
- 본문 컨테이너: 메인과 동일 폭 상한 안에서 **`max-w-3xl`**(`w-full`) — 폼·배지가 한 열로 넉넉히 보이도록 함.
- 카드: **프로필 이모지** 원 + 표시명 + 이메일.
- 폼: **닉네임**, **프로필 이모지**(프리셋 그리드에서만 선택 — 직접 입력 필드 없음), **주·월 거리 목표(km)** — 저장 시 `PATCH /api/user/profile`, 이어서 **`session.update()`** 로 JWT 세션 표시값(닉네임·이모지 등) 갱신(헤더·피드와 즉시 맞춤).
- **배지** 카드(`SettingsBadgesSection`): 획득한 배지(이모지·제목·설명·획득일), **아직 획득 전** 목록(조건 힌트만). 푸시·알림 없음.
- **로그아웃** 버튼(Secondary 스타일) — 클릭 시 세션 종료 후 `/`로 **전체 네비게이션**.
- 하단 `RunLog v0.1 · 토이 프로젝트` 캡션.

### 4.6 랭킹 `/ranking`

- 기간(`week` / `month` / `year` / `all`)·지표(거리·횟수) 칩/쿼리로 전환.
- 목록: 순위·닉네임(또는 미설정 문구)·이모지·수치. (`lib/leaderboard.ts`)

### 4.7 커뮤니티 `/community`

- 상단: 설명, **한 줄 올리기** 폼(`CommunityPostForm`: 내용·챌린지·기록 첨부).
- **피드:** 시간순 단일 목록(챌린지·자유 글 구분 필터 없음).
- 카드(`CommunityPostCard`): 작성자 이모지·닉네임·시간·챌린지 뱃지·본문·첨부 기록 요약.
- 카드 하단: **반응** 프리셋 버튼(로그인 시), **댓글** 영역(로그인 시 입력). (`CommunityPostInteractions`)
  - **피드 SSR**에는 댓글 본문을 싣지 않고 **댓글 건수**만 표시; 카드 마운트 시 클라이언트가 `GET /api/community/posts/[postId]/comments`로 목록을 불러옴.
  - 반응·댓글 제출 후 **전체 `router.refresh()` 없이** API 응답 JSON으로 카운트·내 반응·댓글 목록만 갱신(깜빡임·스크롤 점프 완화).

### 4.8 로그인 영역 라우트 전환 로딩

- `app/(main)/loading.tsx`: 대시보드·랭킹·커뮤니티·기록·설정 간 이동 시 **스켈레톤**(제목·몇 줄 placeholder)으로 체감 대기 완화.

### 5.1 버튼

| 변형 | 용도 |
|------|------|
| **Primary** | Google 로그인, 저장, 추가, FAB |
| **Secondary** | 취소, 로그아웃(테두리 스타일) |
| **Ghost / 링크** | `text-accent` 밑줄 링크(전체 기록 보기 등) |
| **Danger** | 삭제 |

로딩: Primary에 문구 변경(“저장 중…”, “이동 중…”) + 필요 시 `aria-busy`.

### 5.2 카드

- 배경 `surface`, 테두리 `border`, `shadow-card`, 패딩 통일.
- 요약 카드: 라벨 Caption + 큰 숫자(`font-numeric`).

### 5.3 입력

- 라벨 상단, `rounded-input`, 포커스 링 액센트.

### 5.4 차트 (Recharts)

- 컨테이너 높이: **주·월** 약 **260px**(동일 카드 높이로 월 하단 빈 여백 완화), **년·전체** 약 **300px**(`DashboardCharts`).
- `BarChart` **하단 margin** 은 주·월·년·전체 **동일**(코드상 `bottom: 4`, `DashboardCharts`).
- `XAxis` **height**(라벨 대역): 주 **24px**; 월(기울어진 일자 라벨) **30px**; 년·전체(기울어진 월 라벨 등) **44px**. 전체 구간 데이터가 많을 때는 틱 `interval`로 라벨 밀도 조정.
- 툴팁: `--color-surface` 배경, 테두리. **표시할 거리 데이터가 없으면 Tooltip 미렌더.**
- 호버: **`Tooltip`의 기본 커서(회색 영역) 비활성**(`cursor={false}`), **활성 막대 레이어 비활성**(`activeBar={false}`) — 말풍선만 표시.
- 막대 색: `var(--color-chart-primary)`.
