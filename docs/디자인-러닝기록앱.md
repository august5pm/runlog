# 러닝 기록 앱 — UI/UX 디자인 명세

> **구현 동기화 (2026-04):** 실제 화면·토큰·내비와 맞춰 갱신. 토큰 원본은 [design-tokens.css](./design-tokens.css), **런타임 번들**에는 `app/globals.css` 상단 `:root`가 동일 값으로 인라인 복사됨(`@import` 누락 방지). Tailwind는 `tailwind.config.ts`에서 위 CSS 변수를 참조.

기획서: [기획-러닝기록앱.md](./기획-러닝기록앱.md)

## 1. 디자인 방향

| 키워드 | 적용 |
|--------|------|
| **가볍고 빠름** | 여백 넉넉, 한 화면에 정보 단계적 공개 |
| **신뢰·기록** | 숫자(거리·페이스) 가독성 우선, 차트는 보조 |
| **에너지** | 액센트 컬러로 “움직임” 표현, 과한 그라데이션은 지양 |
| **토이·1주** | 다크 모드 토큰만 준비(F-08), **라이트 기본**·토글 UI는 미구현 |

**톤:** 친근한 설명형 문장. 숫자·날짜 라벨은 짧고 명확하게.

## 2. 정보 구조 · 내비게이션

### 2.1 로그인 후 IA

```
대시보드 (/dashboard) — 홈, 요약·주/월 차트·최근 기록
기록 (/runs) — 목록
  └ 신규 (/runs/new)
  └ 수정·삭제 (/runs/[id]/edit)
설정 (/settings) — 프로필, 로그아웃
```

### 2.2 내비 패턴 (구현)

| 뷰포트 | 패턴 |
|--------|------|
| **모바일 (<768px)** | **하단 고정 탭 3개:** `대시보드` · `기록` · `설정`. 콘텐츠 하단은 탭에 가리지 않도록 `pb-20` 여백. **FAB** `+`: 대시보드에서 기록 추가(우하단, `aria-label="기록 추가"`). |
| **태블릿/데스크톱 (md↑)** | **상단 고정 헤더:** 좌측 `RunLog` 로고 링크 → `/dashboard`, 가운데 **텍스트 네비 링크** 3개(대시보드·기록·설정), 우측 **계정 캡슐**(이니셜 원 + 「로그인」라벨 + 이름/이메일 truncate). **아바타 드롭다운은 미사용** — 로그아웃은 **설정** 페이지 버튼으로만 제공. |

디자인 초안의 “앱바 + 아바타 드롭다운”과 달리, 구현은 **동일 정보를 플랫 링크 + 설정 집중**으로 단순화함.

터치 최소 영역: **44×44px** 이상을 목표(버튼·탭·FAB).

## 3. 디자인 토큰 요약

상세 값은 [design-tokens.css](./design-tokens.css) 및 `app/globals.css` 참고.

### 3.1 컬러 (라이트 기본)

| 토큰(CSS) | Tailwind 매핑 예 | 용도 |
|-----------|------------------|------|
| `--color-bg` | `bg-bg` | 페이지 배경 |
| `--color-surface` | `bg-surface` | 카드·시트 배경 |
| `--color-border` | `border-border` | 구분선·입력 테두리 |
| `--color-text` | `text-foreground` | 본문(코드상 `foreground` 명칭) |
| `--color-text-muted` | `text-muted` | 보조 설명·날짜 |
| `--color-text-subtle` | `text-subtle` | 더 약한 메타 |
| `--color-accent` | `bg-accent`, `text-accent` | 주요 CTA·차트 메인 시리즈·활성 세그먼트 |
| `--color-accent-muted` | `bg-accent-muted` | 호버·배지 배경 |
| `--color-danger` | `bg-danger`, `text-danger` | 삭제·치명적 오류 |
| `--color-danger-muted` | `bg-danger-muted` | 인라인 경고 박스 배경(랜딩 OAuth 안내 등) |
| `--color-success` | `text-success` 등 | 저장 완료 등(선택) |

차트: 메인 막대 `--color-chart-primary`(액센트와 동일), 그리드 `--color-chart-grid`. **보조 시리즈(페이스 라인 등)는 현재 차트에 없음** — `--color-chart-secondary`는 확장용.

### 3.2 타이포그래피

| 스타일 | 용도 |
|--------|------|
| **Display** | 랜딩 히어로 한 줄 (`text-display`) |
| **H1** | 페이지 제목 (`text-h1`) |
| **H2** | 섹션 (`text-h2`) |
| **Body** | 본문·폼 라벨 (`text-body`) |
| **Caption** | 메타·단위 (`text-caption`) |
| **Numeric** | 거리·페이스 (`font-numeric`, `tabular-nums` 계열) |

폰트: 시스템 스택 + `"Noto Sans KR"` 등(`globals.css` / 토큰 파일 참고).

### 3.3 간격·모서리

- 베이스 단위: **4px**. 섹션 간 세로: `space-y-6`~`8` 수준.
- 카드: `rounded-card`(`--radius-card` ≈ 12px), 버튼·입력 `rounded-button` / `rounded-input`.

### 3.4 그림자

- 카드: `--shadow-card` → `shadow-card`
- FAB: `--shadow-fab` → `shadow-fab`

## 4. 화면별 와이어프레임 (구현 기준)

### 4.1 랜딩 `/` (비로그인)

- **상단 별도 앱바 없음.** 세로 중앙 정렬 단일 컬럼(max-width 제한).
- 상단 작은 캡션 `RunLog` → 큰 제목 `오늘 러닝, 가볍게 기록` → 부제(거리·시간·메모·주간 차트·Google 안내).
- **Primary:** `Google로 계속하기` 전폭 버튼.
- `GOOGLE_CLIENT_ID` 등 미설정 시 **호박색 안내 카드**로 `.env` 안내.
- OAuth 오류: URL `error` 파라미터 또는 `/auth/error`에서 메시지.

### 4.2 대시보드 `/dashboard`

- 제목 `대시보드` + `안녕하세요, {이름}님`.
- **요약 카드 3개(그리드):**  
  - 이번 주(또는 달) **횟수**  
  - 이번 주(또는 달) **거리(km)**  
  - 이번 주(또는 달) **운동 시간**(합산 초 → `N분 N초`, 없으면 `—`)  
  *(초안의 “총거리” 3카드와 달리, 세 번째는 **시간** 카드로 구현됨.)*
- **차트 섹션:** 부제 `이번 주 거리` / `이번 달 거리` + 캡션(요일별 합산 / 월 제목·일별 합산). 우측(또는 아래) **세그먼트:** `이번 주` | `이번 달`**(`?range=month`)**.
- **해당 기간에 런이 한 건도 없으면:** 막대 차트 대신 **점선 카드 안내 문구**만 표시(“기록을 추가하면 차트가 표시됩니다.”).  
  기간에 런이 있으면 **Recharts 막대**, 거리가 모두 0인 경우는 없음(런 있으면 합산 > 0). **데이터 막대가 없을 때는 툴팁·막대 activeBar 비활성**(구현 세부).
- **최근 기록:** 최대 5건, 행 클릭 → `/runs/[id]/edit`. 거리·페이스(/km), 선택 시 심박·케이던스 한 줄, 메모 clamp. 모바일 상단 `기록 추가` 링크 + FAB `+`.
- **전체 기록 보기:** `md` 이상에서만 “전체 기록 보기 →” 링크 노출(모바일은 하단 탭 `기록`으로 이동).

### 4.3 기록 목록 `/runs`

- 제목 `기록` + 우측 `추가` 버튼 → `/runs/new`.
- 빈 상태: 점선 카드 + “아직 기록이 없습니다. 첫 러닝을 남겨 보세요.”
- 각 행: **카드 전체가 링크** → 수정 화면. 날짜, `km · 분·초 · 페이스 …/km`, 심박·케이던스(있을 때), 메모.  
  **행 우측 `⋯` 메뉴는 없음** — 삭제는 수정 화면에서만.

### 4.4 기록 추가·수정 `/runs/new`, `/runs/[id]/edit`

- 폼 카드: 활동일(date), 거리(km), **시간 분 / 초(0–59)** 그리드, **평균 심박·케이던스(선택)** 그리드, 메모 textarea, 저장·취소. 수정 시 **삭제**(Danger, `confirm` 문구: 기획 마이크로카피와 동일 취지).
- 케이던스 보조 설명 캡션(spm).

### 4.5 설정 `/settings`

- 제목 `설정`.
- 카드: Google 프로필 이미지(또는 이니셜 원) + 이름 + 이메일.
- **로그아웃** 버튼(Secondary 스타일) — 클릭 시 세션 종료 후 `/`로 **전체 네비게이션**.
- 하단 `RunLog v0.1 · 토이 프로젝트` 캡션.

## 5. 컴포넌트 사양

### 5.1 버튼

| 변형 | 용도 |
|------|------|
| **Primary** | Google 로그인, 저장, 추가, FAB |
| **Secondary** | 취소, 로그아웃(테두리 스타일) |
| **Ghost / 링크** | `text-accent` 밑줄 링크(전체 기록 보기 등) |
| **Danger** | 삭제 |

로딩: Primary에 문구 변경(“저장 중…”, “이동 중…”) + 필요 시 `aria-busy`.

### 5.2 카드

- 배경 `surface`, 테두리 `border`, `shadow-card`, 패딩 통일.
- 요약 카드: 라벨 Caption + 큰 숫자(`font-numeric`).

### 5.3 입력

- 라벨 상단, `rounded-input`, 포커스 링 액센트.

### 5.4 차트 (Recharts)

- 컨테이너 높이: 주간 약 260px, 월간 약 300px.
- 툴팁: `--color-surface` 배경, 테두리. **표시할 거리 데이터가 없으면 Tooltip 미렌더·막대 activeBar 비활성.**
- 막대 색: `var(--color-chart-primary)`.
