# 러닝 기록 앱 — 제품 기획서 (토이 / 1주 스프린트)

> **구현 동기화:** 아래 본문은 현재 저장소(Next.js 14 App Router, Prisma, NextAuth) 기준으로 점검·갱신됨. (브랜딩·차트 UX 등 후속 반영 포함)

## 1. 한 줄 정의

**Google로 로그인한 사용자가 러닝 세션을 기록하고, 주간·월간으로 시각화하는 웹 앱.**

---

## 2. 목표와 비목표

### 2.1 목표 (1주 내 달성)

- 로그인 후 **나의 러닝 기록 CRUD**가 가능하다.
- **대시보드**에서 최근 기록과 **주·월 단위 막대 차트(거리)**를 본다.
- **Vercel**에 배포되어 본인 계정으로 end-to-end 검증이 가능하다.

### 2.2 비목표 (토이 범위 밖 또는 미구현)

- 소셜 **팔로우·DM·알림** 등 관계형 소셜 기능
- 웨어러블/GPX 자동 연동(나중 확장 후보)
- 네이티브 앱
- 결제·팀·권한 세분화
- 외부 LLM·생성형 AI 요약·코칭

> **참고:** 기간·지표별 **랭킹**(`/ranking`)과 **커뮤니티 피드**(`/community`, 댓글·반응)는 구현되어 있으나, 팔로우·비공개 그룹 등은 포함하지 않는다.

## 3. 타깃 사용자와 시나리오

| 구분 | 설명 |
|------|------|
| 주 사용자 | 본인 1인 또는 소수 베타(토이) |
| 핵심 시나리오 A | 러닝 직후 또는 그날 중에 **거리·시간·메모**를 남긴다. |
| 핵심 시나리오 B | 주말에 **이번 주(또는 달) 합산 거리·횟수·시간**과 일별 거리 차트를 본다. 대시보드 상단에서 **당일 날씨 요약**(기온·습도·강수, 외부 날씨 링크)을 함께 본다. |
| 핵심 시나리오 C | 메모로 그날 컨디션·메모를 남긴다. (선택: 심박·케이던스) |
| 핵심 시나리오 D (선택) | **설정**에서 획득한 **배지**를 모아 본다. 조건은 주·월 런 횟수·연속 기록 일수 등(`lib/badges.ts`). |

## 4. 기능 범위

### 4.1 MVP (필수) — 구현 상태

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | Google 로그인 / 로그아웃 | NextAuth.js, JWT 세션 + Prisma 어댑터. OAuth 미설정 시 랜딩에서 버튼 비활성·안내. 오류 시 `/auth/error` 또는 `/?error=` 연동. **세션 콜백**은 JWT의 닉네임·이모지 등을 사용(매 요청 `User` 테이블 조회 없음). 프로필 저장 시 클라이언트 `session.update()`로 JWT 갱신 |
| F-02 | 러닝 기록 목록 | 본인 데이터만, 날짜 역순(최대 100건 화면) |
| F-03 | 기록 추가·수정·삭제 | 폼 유효성 검사. **선택 필드:** 평균 심박(bpm), 케이던스(spm). 삭제는 수정 화면에서 확인 후 실행 |
| F-04 | 대시보드 | 최근 5건, **연속 기록 일수**·**주·월 거리 목표** 카드, 기간별(`주`·`월`·`년`·`전체`, `?range=`) 요약 카드 **횟수·거리·운동 시간**. Recharts **막대 그래프**(주: 요일별, 월: 일별, 년: 월별, 전체: 월별 합산). 페이스는 **목록/최근 기록 텍스트**로만 표시. 차트 호버 시 **툴팁만**(회색 커서 밴드·활성 막대 오버레이 없음) |

### 4.2 확장·선택 (구현된 항목)

| ID | 기능 | 비고 |
|----|------|------|
| F-06 | 기간별 차트 | 대시보드 세그먼트 `주`(기본) / `월`(`?range=month`) / `년`(`?range=year`) / `전체`(`?range=all`) |
| F-07 | 날씨 스냅샷 | 공공데이터포털 **기상청 단기예보 API**(`VilageFcstInfoService_2.0`): 초단기실황 + 초단기예보로 기온·습도·PTY·SKY. 서버 `getWeatherSnapshot()` → 대시보드 **제목·인사 행 우측**(모바일·데스크톱 동일). UI: 이모지+수치 가로 배치, 외부 **`네이버 날씨 →`** 링크(모바일은 텍스트 열 왼쪽·데스크톱 전폭 우측 등 반응형). 격자·관측 슬롯 기준 **`unstable_cache`(revalidate 약 180초)** 로 API 호출 완화. **이미지 URL 없음** — `weather-icon-map`·**이모지**(`WeatherIcon`). 키 미설정 시 안내 메시지. **체감 속도:** 날씨 영역은 `Suspense`+스켈레톤으로 스트리밍, `React.cache`로 동일 요청 내 조회 1회 공유 |
| F-08 | 다크 모드 | `globals.css` / `design-tokens.css`에 `[data-theme="dark"]` 토큰만 준비. **UI 토글은 미구현**(라이트 기본) |
| F-09 | 랭킹 | `/ranking` — 기간(`week`/`month`/`year`/`all`)·지표(거리·횟수)별 리더보드, 닉네임·이모지 표시 |
| F-10 | 커뮤니티 | `/community` — `Challenge`·`CommunityPost`(한 줄 280자, 선택 `runId`·`challengeId`), `CommunityComment`, `CommunityPostReaction`(프리셋 kind, 글당 사용자 1개). 피드는 **닉네임** 위주 표시. **피드 Prisma 쿼리**는 댓글 본문을 포함하지 않고 `_count.comments`만 사용; 댓글은 클라이언트가 `GET .../comments`로 로드. 반응/댓글 작성 후 **페이지 전체 `refresh` 없이** API JSON으로 카운트·목록 갱신. 챌린지 롤링 후 세션·챌린지 목록·피드 조회는 **`Promise.all` 병렬** |
| F-11 | 프로필·목표 | 설정에서 닉네임·프로필 이모지(프리셋)·`weeklyDistanceGoalKm` / `monthlyDistanceGoalKm`. `PATCH /api/user/profile` |
| F-12 | 배지(가벼운 버전) | 조건 충족 시 `UserBadge`에 1회 저장(획득 시각 유지, 기록 삭제로 회수하지 않음). **설정**에서만 목록·미획득 힌트 표시, 푸시 없음. 동기화: `POST`/`PATCH` `/api/runs` 성공 후·설정 페이지 로드 시 `syncUserBadges`. 종류·이모지: `lib/badges.ts`의 `ALL_BADGES`(예: 첫 기록 🎽, 주 3회 🎯, 주 5회 🔥, 월 10회 🏅, 연속 7일 ✨, 연속 14일 🏆) |

## 5. 데이터 모델 (Prisma 관점)

### 5.1 엔티티

**User** (NextAuth 연동)

- `id` (cuid)
- `email`, `name`, `image` (OAuth — `image`는 DB에 보관하나 UI에서는 프로필 이모지 사용)
- `nickname`, `profileEmoji` (선택): 헤더·설정·랭킹·커뮤니티 표시. 미설정 시 이름·이메일 앞부분 / 기본 이모지 🏃
- `weeklyDistanceGoalKm`, `monthlyDistanceGoalKm` (선택): 주·월 거리 목표(km), 대시보드 달성률
- `emailVerified`, Account / Session / VerificationToken 관계
- 커뮤니티 관계: `communityPosts`, `communityComments`, `communityReactions`(`CommunityPostReaction`)
- `badges` → `UserBadge[]`

**Run** (러닝 세션)

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| id | String (cuid) | O | |
| userId | String | O | User FK |
| date | DateTime | O | 활동일(폼에서 로컬 날짜 + 정오 근처 UTC 저장) |
| distanceKm | Decimal | O | km |
| durationSec | Int | O | 초 |
| avgHeartRate | Int? | X | 평균 심박(bpm), 선택 |
| cadence | Int? | X | 케이던스(spm), 선택 |
| notes | String? | X | 메모 |
| createdAt / updatedAt | DateTime | O | |

**파생 값**

- **페이스(분/km)**: DB 컬럼 없음, `durationSec`·`distanceKm`로 계산.

**Challenge** (기간형 챌린지, slug로 idempotent)

- `id`, `slug`(unique), `title`, `summary?`, `startsAt`, `endsAt` — 커뮤니티 글과 연결

**CommunityPost**

- `userId`, `challengeId?`, `runId?`, `content`(최대 280자), `createdAt` — 본인 `Run`만 `runId` 첨부 가능(API 검증)

**CommunityComment** / **CommunityPostReaction**

- 댓글: `postId`, `userId`, `content`(최대 200자)
- 반응: 복합 PK `(postId, userId)`, `kind`(프리셋 문자열)

**UserBadge**

- `userId`, `badgeId`(문자열 슬러그, 예: `first_run`, `week_3`), `earnedAt`
- `@@unique([userId, badgeId])` — 동일 배지 중복 저장 방지

### 5.2 인덱스

- `Run`: `(userId, date DESC)` — 목록·주간 집계에 유리

## 6. API / Route Handlers

대부분 인증 필요. Run·프로필·반응·댓글 POST 등은 세션의 `user.id` 검증. **`GET /api/community/posts/[postId]/comments`** 는 글이 있으면 로그인 없이 조회 가능(공개 피드용).

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/runs` | 쿼리: `from`, `to`(선택), `limit`(기본 30, 1~100). 날짜 내림차순 |
| POST | `/api/runs` | 생성(JSON: date, distanceKm, durationSec, notes?, avgHeartRate?, cadence?) |
| GET | `/api/runs/[id]` | 단건 |
| PATCH | `/api/runs/[id]` | 수정 |
| DELETE | `/api/runs/[id]` | 삭제 |
| PATCH | `/api/user/profile` | 닉네임·이모지·주/월 목표 등 |
| POST | `/api/community/posts` | 한 줄 글 (`content`, `challengeId?`, `runId?`) |
| GET | `/api/community/posts/[postId]/comments` | 해당 글 댓글 목록(상한 있음)·`total` 건수 |
| POST | `/api/community/posts/[postId]/comments` | 댓글(응답에 생성된 댓글 객체 포함) |
| POST | `/api/community/posts/[postId]/reactions` | 반응 토글/교체(프리셋 `kind`, 응답에 `reactionCounts`·`myReactionKind` 포함) |

배지는 **별도 공개 API 없음**. 기록 생성·수정 처리 및 설정 페이지에서 서버가 `syncUserBadges`를 호출한다.

에러 응답: JSON `{ error: string }`, 적절한 HTTP 상태 코드.

## 9. 기술 스택 (확정안)

| 영역 | 선택 |
|------|------|
| 프론트 | Next.js 14(App Router), Tailwind CSS, Recharts |
| API | Next.js Route Handlers |
| DB | PostgreSQL (Supabase) |
| ORM | Prisma |
| 인증 | NextAuth.js + Google OAuth |
| 배포 | Vercel |

### 9.1 Supabase + Prisma 운영 메모

- **마이그레이션**: Direct connection URL 사용.
- **앱 런타임(Vercel)**: Pooling URL + Prisma 문서 권장 파라미터 준수.
- 환경 변수 예: `DATABASE_URL`(pool), `DIRECT_URL`(migrate).

### 9.2 기상청 API(날씨)

- **필수:** `DATA_GO_KR_SERVICE_KEY`(공공데이터포털 일반 인증키, Encoding 권장).
- **선택:** `WEATHER_LAT` / `WEATHER_LON`(격자 변환), `WEATHER_GRID_NX` / `WEATHER_GRID_NY`(격자 직접 지정), `NAVER_WEATHER_URL`(외부 링크 대상).
- 포털에서 **단기예보 조회(VilageFcstInfoService)** 활용 신청이 되어 있어야 초단기실황·초단기예보 호출이 가능함.
- 앱에서는 동일 격자·관측 시각 슬롯에 대해 Next **`unstable_cache`**(짧은 TTL)로 호출 빈도를 줄임. 캐시된 값은 실시간 관측과 수분 차이 날 수 있음.

### 9.3 체감 성능(구현 메모)

- 로그인 영역(`app/(main)`) 라우트 전환 시 **`loading.tsx`** 스켈레톤.
- 커뮤니티 피드·반응/댓글 상호작용은 위 F-10·API 절 참고.
