# 러닝 기록 앱 — 제품 기획서 (토이 / 1주 스프린트)

> **구현 동기화:** 아래 본문은 현재 저장소(Next.js 14 App Router, Prisma, NextAuth) 기준으로 점검·갱신됨. (브랜딩·차트 UX 등 후속 반영 포함)

## 1. 한 줄 정의

**Google로 로그인한 사용자가 러닝 세션을 기록하고, 주간·월간으로 시각화하는 웹 앱.**

---

## 2. 목표와 비목표

### 2.1 목표 (1주 내 달성)

- 로그인 후 **나의 러닝 기록 CRUD**가 가능하다.
- **대시보드**에서 최근 기록과 **주·월 단위 막대 차트(거리)**를 본다.
- **Vercel**에 배포되어 본인 계정으로 end-to-end 검증이 가능하다.

### 2.2 비목표 (이번 주에는 하지 않음)

- 소셜·팔로우·리더보드
- 웨어러블/GPX 자동 연동(나중 확장 후보)
- 네이티브 앱
- 결제·팀·권한 세분화
- 외부 LLM·생성형 AI 요약·코칭

---

## 3. 타깃 사용자와 시나리오

| 구분 | 설명 |
|------|------|
| 주 사용자 | 본인 1인 또는 소수 베타(토이) |
| 핵심 시나리오 A | 러닝 직후 또는 그날 중에 **거리·시간·메모**를 남긴다. |
| 핵심 시나리오 B | 주말에 **이번 주(또는 달) 합산 거리·횟수·시간**과 일별 거리 차트를 본다. 대시보드 상단에서 **당일 날씨 요약**(기온·습도·강수, 외부 날씨 링크)을 함께 본다. |
| 핵심 시나리오 C | 메모로 그날 컨디션·메모를 남긴다. (선택: 심박·케이던스) |

## 4. 기능 범위

### 4.1 MVP (필수) — 구현 상태

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | Google 로그인 / 로그아웃 | NextAuth.js, JWT 세션 + Prisma 어댑터. OAuth 미설정 시 랜딩에서 버튼 비활성·안내. 오류 시 `/auth/error` 또는 `/?error=` 연동 |
| F-02 | 러닝 기록 목록 | 본인 데이터만, 날짜 역순(최대 100건 화면) |
| F-03 | 기록 추가·수정·삭제 | 폼 유효성 검사. **선택 필드:** 평균 심박(bpm), 케이던스(spm). 삭제는 수정 화면에서 확인 후 실행 |
| F-04 | 대시보드 | 최근 5건, 요약 카드 3종(기간별 **횟수·거리·운동 시간**). Recharts **막대 그래프: 일·요일별 거리(km)**. 페이스는 **목록/최근 기록 텍스트**로만 표시(별도 페이스 추이 차트 없음). 차트 호버 시 **툴팁만** 표시(기본 회색 커서 밴드·활성 막대 오버레이 없음) |

### 4.2 확장·선택 (구현된 항목)

| ID | 기능 | 비고 |
|----|------|------|
| F-06 | 월간 보기 | 대시보드에서 `이번 주` / `이번 달` 세그먼트(`?range=month`)로 전환, 달력 월 일별 합산 막대 차트 |
| F-07 | 날씨 스냅샷 | 공공데이터포털 **기상청 단기예보 API**(`VilageFcstInfoService_2.0`): 초단기실황 + 초단기예보로 기온·습도·강수형태(PTY)·하늘상태(SKY). 서버에서 `getWeatherSnapshot()` 후 헤더 우측에 표시. **이미지 URL은 API에 없음** — 코드 매핑 SVG 아이콘 사용. 키 미설정 시 안내 메시지 |
| F-08 | 다크 모드 | `globals.css` / `design-tokens.css`에 `[data-theme="dark"]` 토큰만 준비. **UI 토글은 미구현**(라이트 기본) |

## 5. 데이터 모델 (Prisma 관점)

### 5.1 엔티티

**User** (NextAuth 연동)

- `id` (cuid)
- `email`, `name`, `image` (OAuth)
- `emailVerified`, Account / Session / VerificationToken 관계

**Run** (러닝 세션)

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| id | String (cuid) | O | |
| userId | String | O | User FK |
| date | DateTime | O | 활동일(폼에서 로컬 날짜 + 정오 근처 UTC 저장) |
| distanceKm | Decimal | O | km |
| durationSec | Int | O | 초 |
| avgHeartRate | Int? | X | 평균 심박(bpm), 선택 |
| cadence | Int? | X | 케이던스(spm), 선택 |
| notes | String? | X | 메모 |
| createdAt / updatedAt | DateTime | O | |

**파생 값**

- **페이스(분/km)**: DB 컬럼 없음, `durationSec`·`distanceKm`로 계산.

### 5.2 인덱스

- `Run`: `(userId, date DESC)` — 목록·주간 집계에 유리

## 6. API / Route Handlers

인증된 요청만 허용. 세션의 `user.id`와 `Run.userId` 일치 검증.

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/runs` | 쿼리: `from`, `to`(선택), `limit`(기본 30, 1~100). 날짜 내림차순 |
| POST | `/api/runs` | 생성(JSON: date, distanceKm, durationSec, notes?, avgHeartRate?, cadence?) |
| GET | `/api/runs/[id]` | 단건 |
| PATCH | `/api/runs/[id]` | 수정 |
| DELETE | `/api/runs/[id]` | 삭제 |

에러 응답: JSON `{ error: string }`, 적절한 HTTP 상태 코드.

## 7. 화면 구성 (정보 구조)

| 경로 | 설명 |
|------|------|
| `/` | 랜딩: **`public/logo.png` 기반 로고 이미지 + RunLog 워드마크**, 히어로·부제(부제는 줄바꿈). Google 로그인. 세션 있으면 `/dashboard` 리다이렉트. OAuth 오류 시 `error` 쿼리로 메시지 표시 가능 |
| `/auth/error` | NextAuth 커스텀 오류 페이지 |
| `/dashboard` | 헤더 행(제목·인사 + 우측 날씨) + 요약 카드 + 주·월 차트 + 최근 기록. `?range=month` 로 월간 |
| `/runs` | 전체 목록, 추가 버튼 |
| `/runs/new` | 신규 입력 |
| `/runs/[id]/edit` | 수정·삭제 |
| `/settings` | 프로필(아바타·이름·이메일), 로그아웃, 앱 이름/버전 문구 |

**모바일 우선:** 하단 탭 3개(대시보드·기록·설정), 데스크톱은 상단 헤더 + 동일 링크.

## 8. AI / LLM

이번 스코프에서는 **외부 LLM·생성형 AI를 사용하지 않는다.** 주간·월간 합산·차트·페이스 등은 DB와 서버 연산만으로 표시한다.

## 9. 기술 스택 (확정안)

| 영역 | 선택 |
|------|------|
| 프론트 | Next.js 14(App Router), Tailwind CSS, Recharts |
| API | Next.js Route Handlers |
| DB | PostgreSQL (Supabase) |
| ORM | Prisma |
| 인증 | NextAuth.js + Google OAuth |
| 배포 | Vercel |

### 9.1 Supabase + Prisma 운영 메모

- **마이그레이션**: Direct connection URL 사용.
- **앱 런타임(Vercel)**: Pooling URL + Prisma 문서 권장 파라미터 준수.
- 환경 변수 예: `DATABASE_URL`(pool), `DIRECT_URL`(migrate).

### 9.2 기상청 API(날씨)

- **필수:** `DATA_GO_KR_SERVICE_KEY`(공공데이터포털 일반 인증키, Encoding 권장).
- **선택:** `WEATHER_LAT` / `WEATHER_LON`(격자 변환), `WEATHER_GRID_NX` / `WEATHER_GRID_NY`(격자 직접 지정), `NAVER_WEATHER_URL`(외부 링크 대상).
- 포털에서 **단기예보 조회(VilageFcstInfoService)** 활용 신청이 되어 있어야 초단기실황·초단기예보 호출이 가능함.
